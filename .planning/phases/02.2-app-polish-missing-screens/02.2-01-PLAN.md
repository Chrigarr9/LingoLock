  <name>Task 1: Add new-words-per-day storage functions + daily tracking (both adapters)</name>
  <files>src/services/storage.ts, src/services/storage.web.ts</files>
  <action>
Add four new exported functions to storage.ts (using statsStorage MMKV instance) and their parallel implementations in storage.web.ts (using localStorage with 'll.' prefix):

1. `loadNewWordsPerDay(): number` — Returns the configured daily new-word limit. Default: 20. Read from `statsStorage.getNumber('new_words_per_day')` (MMKV) / `localStorage.getItem('ll.new_words_per_day')` (web).

2. `saveNewWordsPerDay(n: number): void` — Clamp to range [1, 50] with `Math.max(1, Math.min(50, n))` before saving.

3. `loadNewWordsIntroducedToday(): number` — Returns how many new words have been introduced today. Check stored date (`new_words_today_date` key) against `new Date().toISOString().slice(0, 10)`. If date differs (new day), return 0. Otherwise return stored count (`new_words_today` key) or 0.

4. `recordNewWordsIntroduced(count: number): void` — Records that `count` new words were introduced. Sets date to today, adds `count` to existing today count (using loadNewWordsIntroducedToday() to get current). This is called from challenge.tsx at session completion, NOT from buildSession (keeps buildSession pure/idempotent per RESEARCH.md Pitfall 2 recommendation).

MMKV keys: `new_words_per_day`, `new_words_today`, `new_words_today_date` (all on statsStorage instance).
Web keys: `ll.new_words_per_day`, `ll.new_words_today`, `ll.new_words_today_date`.

CRITICAL: Both files must export identical function signatures. The web adapter uses `localStorage.getItem()` + `parseInt()` for numbers, `localStorage.setItem()` + `String()` for writes.
  </action>

  <files>src/services/cardSelector.ts</files>
  <action>
Modify `buildSession` to accept and respect a daily new-word budget:

1. Change signature from `buildSession(cardCount: number, _sourceApp?: string)` to `buildSession(dailyNewWordBudget: number, _sourceApp?: string)`. The parameter name change reflects its new meaning — it's no longer a fixed card count but a daily cap on new words. Due reviews are unlimited.

2. Add import of `loadNewWordsIntroducedToday` from `./storage`.

3. New session composition logic:
   - Collect ALL due review cards (not limited by cardCount anymore — continuous sessions review everything due).
   - Collect new words from current chapter first, then subsequent chapters.
   - Calculate remaining budget: `const remainingBudget = Math.max(0, dailyNewWordBudget - loadNewWordsIntroducedToday())`.
   - Take at most `remainingBudget` new words: `const selectedNew = newCards.slice(0, remainingBudget)`.
   - Combine: `[...shuffle(selectedDue), ...selectedNew]` — shuffle due reviews for variety but keep new words after reviews.
   - Remove the old `maxDue = cardCount - 1` / `remainingSlots` logic entirely.

4. NOTE: Do NOT call `recordNewWordsIntroduced` inside buildSession. This function must remain pure/idempotent. Recording happens in challenge.tsx at session completion (Plan 02.2-03 handles this).

5. If `dailyNewWordBudget` is 0 (edge case: user set to 0 somehow, or passed from old deep links), treat as "no new words, reviews only".

6. Update the JSDoc to reflect the new parameter meaning and that sessions are now variable-length (all due + capped new).
