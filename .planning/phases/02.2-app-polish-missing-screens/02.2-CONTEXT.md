<decisions>
## Implementation Decisions

### Missing Screens
- Build all three screens: Vocabulary browser, Stats detail, Settings
- Each screen is a full route (new page with back navigation), matching existing Stack pattern in `_layout.tsx`
- Hide tutorial link on web (Platform check) — the iOS Shortcuts tutorial is irrelevant for PWA users

### Vocabulary Browser
- Grouped by chapter with section headers — matches the chapter-based learning progression
- Each word row shows: Spanish word, German translation, mastery status indicator (new/learning/mastered)
- Tapping a word opens a detail view: full sentence, translation, image (if available), FSRS status, chapter
- Search bar at top to filter by word or translation

### Stats Detail Screen
- Per-chapter breakdown: mastery % for each chapter, total cards vs mastered, cards due per chapter
- Visual style: progress bars + big numbers, consistent with existing glass-surface design (no charting library)
- Tapping a chapter row navigates to vocabulary list filtered to that chapter (connects the two new screens)

### Session Flow (Major Change)
- Practice sessions are continuous — no fixed card count, no session "end"
- User practices until all due cards are reviewed AND all daily new cards are introduced
- When that milestone is reached: show richer celebration (accuracy percentage, streak update like "3 day streak!", motivational message based on score)
- User can go back to home screen at any time during practice (voluntary exit, not an abort)
- "Start Practice" CTA from home goes into this continuous flow
- Future: forced sessions (unlock/app-open) will have fixed counts, but not needed yet

### New Words Per Day
- Limit of 20 new words introduced per day (setting, configurable)
- This controls pacing — due reviews are unlimited, but new card introduction is capped
- Once all due + daily new are done → celebration screen

### Settings Screen
- Minimal: mute toggle (audio on/off default) + new vocabulary per day setting (default 20)
- Persisted in MMKV (mute already uses MMKV, new-per-day setting follows same pattern)

### Empty State (0 Cards Due)
- Allow practice anyway — relabel CTA to "Review anyway" so users can practice even with nothing due
- Keep encouraging tone

### Home Screen Greeting
- Time-based greeting in Spanish: "Buenos días" / "Buenas tardes" / "Buenas noches" based on time of day
- Spanish-learning-relevant, zero complexity

### Claude's Discretion
- Exact mastery indicator design (dot, badge, icon, color)
- Vocabulary detail view layout
- Celebration screen motivational messages
- Stats screen layout and spacing
- Settings screen visual design
- Time thresholds for greeting (morning/afternoon/evening cutoffs)
- "Review anyway" button styling when 0 due

</decisions>

<specifics>
## Specific Ideas

- Session flow: continuous practice is the core change — the "Start Practice" button enters an open-ended flow, not a 5-card burst
- Celebration only triggers on the meaningful milestone (all due + all daily new done), not on arbitrary card counts
- The 20 new-words-per-day default paces learning so users aren't overwhelmed
- Stats ↔ Vocabulary cross-navigation: tapping a chapter in stats opens vocab filtered to that chapter
