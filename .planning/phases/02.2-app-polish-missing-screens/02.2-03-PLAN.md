  <name>Task 1: Refactor challenge.tsx to continuous session flow with richer celebration</name>
  <files>app/challenge.tsx</files>
  <action>
Refactor challenge.tsx from fixed-count sessions to continuous sessions with daily new-word tracking and richer celebration.

**1. Add mode param (per RESEARCH.md Pitfall 4):**
Update params interface to include `mode`:
```typescript
const params = useLocalSearchParams<{
  source: string;
  count: string;
  type: 'unlock' | 'app_open';
  mode?: 'continuous' | 'fixed';
}>();
```
Default `mode` to `'continuous'` when absent.

**2. Session initialization (useEffect):**
- Import `loadNewWordsPerDay` and `recordNewWordsIntroduced` from storage.
- When `mode === 'continuous'`: call `buildSession(loadNewWordsPerDay(), params.source)`. The new buildSession returns a variable-length array (all due reviews + capped new words).
- When `mode === 'fixed'`: call `buildSession(parseInt(params.count || '3', 10), params.source)` for forward-compat with future forced sessions.
- Set `originalCardCount.current = session.length` (dynamic, based on what buildSession returned).
- Track which cards are new at session start for later recording:
  ```typescript
  const newCardCount = useRef(0);
  // In useEffect after building session:
  newCardCount.current = session.filter(sc => loadCardState(sc.card.id) === null).length;
  ```

**3. Session completion (advanceToNext):**
- Completion condition remains: `nextIndex >= originalCardCount.current` (wrong-answer re-insertions extend queue past original count but do NOT change completion point — same as current behavior).
- In the completion branch, AFTER `updateStatsAfterSession(...)`, call `recordNewWordsIntroduced(newCardCount.current)` to persist the daily new-word count.

**4. Abort handling:**
- Continuous practice exits (mode === 'continuous') should NOT call `recordAbort`. Only forced sessions (mode === 'fixed' AND type === 'unlock') should.
- Update the close button onPress:
  ```typescript
  if (params.mode === 'fixed' && !isComplete) {
    recordAbort(params.source ?? 'unknown');
  }
  ```

**5. Richer celebration screen:**
Replace the current minimal completion UI with enriched celebration:

```typescript
// Compute celebration data
const accuracyPercent = Math.round((correctCount / (originalCardCount.current || 1)) * 100);
const streakCount = getStreak(); // import from statsService

function getMotivationalMessage(accuracy: number): string {
  if (accuracy === 100) return '!Perfecto! Every answer correct.';
  if (accuracy >= 80) return '!Muy bien! Great session.';
  if (accuracy >= 60) return '!Bien! Keep practising.';
  return 'Every mistake is a lesson. !Animo!';
}
```

Celebration card layout:
- Big accuracy percentage (e.g., "92%") as hero number
- Subtitle: "{correctCount}/{originalCardCount.current} correct answers"
- Motivational message below
- If `streakCount > 1`: show streak line (e.g., "3 day streak!" with fire icon)
- "Done" button that calls `router.back()` (replaces ContinueButton which was for deep-link return flow)
- Keep ContinueButton only when `params.type === 'unlock'` or `params.type === 'app_open'` AND `mode === 'fixed'` (for future forced sessions)

**6. Remove hardcoded count:**
- Remove `const cardCount = Math.min(parseInt(params.count || '3', 10), 10);` line (no longer used in continuous mode).
- Keep params.count available for fixed mode only.

**7. Handle empty session:**
- If `buildSession` returns an empty array (no due cards and daily budget exhausted), show a "You're all caught up!" message instead of blank screen. Use `isComplete = true` immediately with a "no cards" variant of the completion screen.
