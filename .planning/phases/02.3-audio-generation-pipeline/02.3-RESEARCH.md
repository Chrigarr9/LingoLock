## Summary

Researched implementation patterns for extending the Spanish content pipeline with AI text-to-speech generation. The standard approach involves using official TTS provider SDKs (Google Cloud TTS or OpenAI TTS), implementing manifest-based incremental generation to avoid redundant API calls, and following the existing image generation pipeline patterns.

Google Cloud TTS offers superior Spanish voice quality with 380+ voices across 75+ languages, better pronunciation accuracy (81.97% vs 77.30%), and more neutral accent options compared to OpenAI. OpenAI TTS defaults to Latin American Spanish accents which may not match the "neutral Spanish" requirement. Both providers support MP3 output at configurable bitrates, with 64kbps mono being optimal for voice-only content (AM radio quality, ~15KB per sentence).

The established architecture pattern mirrors the existing image generation pass: manifest-based caching with content hashing, graceful error handling (skip failed sentences), optional execution (check API keys), and parallel multi-provider support configured via YAML. Build integration follows the same pattern as images: copy audio files to assets directory, generate static require() map for Metro bundler, add optional audioUrl field to cards.

**Primary recommendation:** Use Google Cloud TTS as primary provider for better Spanish quality, implement manifest pattern identical to image generation, mirror build-content.ts image integration pattern for audio files.

### Pattern 1: Multi-Provider TTS Generator (Mirror ImageGenerator)

**What:** Python class that abstracts Google Cloud TTS and OpenAI TTS APIs with identical interface pattern as ImageGenerator.

**When to use:** Pass 6 of content pipeline, standalone audio generation script.

**Example:**
```python
# Source: Derived from pipeline/image_generator.py pattern + official TTS docs
from pathlib import Path
from pipeline.config import DeckConfig
from pipeline.models import AudioManifest, AudioManifestEntry, SentencePair
import hashlib
import json

class AudioGenerator:
    def __init__(
        self,
        config: DeckConfig,
        api_key: str,
        output_base: Path | None = None,
        max_retries: int = 3,
    ):
        self._config = config
        self._api_key = api_key
        self._output_base = output_base or Path("output")
        self._max_retries = max_retries
        self._audio_config = config.audio_generation

        # Initialize provider-specific client
        if self._audio_config.provider == "google":
            from google.cloud import texttospeech
            self._client = texttospeech.TextToSpeechClient(
                client_options={"api_key": api_key}
            )
        elif self._audio_config.provider == "openai":
            from openai import OpenAI
            self._client = OpenAI(api_key=api_key)

    def _deck_dir(self) -> Path:
        return self._output_base / self._config.deck.id

    def _sentence_key(self, chapter: int, sentence_index: int) -> str:
        """Same format as images: ch{NN}_s{NN}"""
        ch = str(chapter).zfill(2)
        si = str(sentence_index).zfill(2)
        return f"ch{ch}_s{si}"

    def _content_hash(self, text: str) -> str:
        """Hash sentence text for change detection"""
        return hashlib.sha256(text.encode('utf-8')).hexdigest()[:16]

    def generate_sentence_audio(
        self,
        sentence: SentencePair,
    ) -> AudioManifestEntry:
        """Generate single sentence audio. Returns manifest entry."""
        key = self._sentence_key(sentence.chapter, sentence.sentence_index)
        rel_path = f"audio/{key}.mp3"
        abs_path = self._deck_dir() / rel_path

        try:
            if self._audio_config.provider == "google":
                audio_bytes = self._generate_google(sentence.source)
            else:
                audio_bytes = self._generate_openai(sentence.source)

            abs_path.parent.mkdir(parents=True, exist_ok=True)
            abs_path.write_bytes(audio_bytes)

            return AudioManifestEntry(
                file=rel_path,
                status="success",
                content_hash=self._content_hash(sentence.source),
            )
        except Exception as e:
            return AudioManifestEntry(
                file=None,
                status="failed",
                error=str(e),
                content_hash=self._content_hash(sentence.source),
            )

    def generate_all(self, sentences: list[SentencePair]) -> AudioManifest:
        """Generate all audio. Resumes from existing manifest if present."""
        manifest_path = self._deck_dir() / "audio_manifest.json"

        # Load existing manifest for resumability (same pattern as images)
        existing_audio: dict[str, AudioManifestEntry] = {}
        if manifest_path.exists():
            data = json.loads(manifest_path.read_text())
            for key, entry_data in data.get("audio", {}).items():
                entry = AudioManifestEntry(**entry_data)
                # Keep if successful, file exists, and content unchanged
                if entry.status == "success" and entry.file:
                    abs_path = self._deck_dir() / entry.file
                    if abs_path.exists():
                        existing_audio[key] = entry

        # Generate missing/changed audio
        all_audio = dict(existing_audio)
        for sentence in sentences:
            key = self._sentence_key(sentence.chapter, sentence.sentence_index)
            content_hash = self._content_hash(sentence.source)

            # Skip if unchanged
            if key in existing_audio:
                if existing_audio[key].content_hash == content_hash:
                    continue  # Sentence text unchanged, reuse

            print(f"    Generating {key}...", end=" ", flush=True)
            entry = self.generate_sentence_audio(sentence)
            all_audio[key] = entry
            print(entry.status)

        # Write manifest
        manifest = AudioManifest(
            provider=self._audio_config.provider,
            language=self._config.languages.target_code,
            audio=all_audio,
        )
        manifest_path.write_text(
            json.dumps(manifest.model_dump(), ensure_ascii=False, indent=2)
        )

        return manifest
```

### Pattern 2: Google Cloud TTS Integration

**What:** Spanish male voice, MP3 format, 64kbps bitrate.

**When to use:** Primary provider in audio_generator.py.

**Example:**
```python
# Source: Google Cloud TTS docs + web search findings
def _generate_google(self, text: str) -> bytes:
    """Call Google Cloud TTS. Returns MP3 bytes."""
    from google.cloud import texttospeech

    synthesis_input = texttospeech.SynthesisInput(text=text)

    # Spanish male voice, neutral accent
    voice = texttospeech.VoiceSelectionParams(
        language_code=self._config.languages.target_code,  # e.g., "es-ES"
        ssml_gender=texttospeech.SsmlVoiceGender.MALE,
    )

    # MP3 at 64kbps mono
    audio_config = texttospeech.AudioConfig(
        audio_encoding=texttospeech.AudioEncoding.MP3,
        speaking_rate=1.0,  # Normal speed
        pitch=0.0,          # Neutral pitch
    )

    response = self._client.synthesize_speech(
        input=synthesis_input,
        voice=voice,
        audio_config=audio_config,
    )

    return response.audio_content
```

```python
# Source: OpenAI TTS API docs + web search findings
def _generate_openai(self, text: str) -> bytes:
    """Call OpenAI TTS. Returns MP3 bytes."""
    from openai import OpenAI

    # Note: OpenAI voices are English-optimized but support Spanish
    # Alloy and Nova are popular choices for educational content
    response = self._client.audio.speech.create(
        model="gpt-4o-mini-tts",  # Latest TTS model
        voice="alloy",  # Male-sounding voice
        input=text,
        response_format="mp3",
    )

    # Stream to bytes
    audio_bytes = b""
    for chunk in response.iter_bytes():
        audio_bytes += chunk

    return audio_bytes
```

### Anti-Patterns to Avoid

- **Hand-rolling retry logic:** Use httpx's built-in retry with exponential backoff or tenacity library, don't write custom retry loops.
- **Blocking on audio failures:** If TTS API fails for one sentence, skip it and continue (like images), never fail entire pipeline.
- **Regenerating unchanged audio:** Always check content hash before calling TTS API to avoid redundant costs.
- **Hardcoding language:** Read from config.languages.target_code, not hardcoded "es-ES".
- **Using SSML prematurely:** Start with plain text TTS, only add SSML if specific pronunciation issues arise (YAGNI principle).
- **Stereo audio for voice:** Use mono to cut file size in half with no quality loss for speech.
- **High bitrates for voice:** 64kbps is AM radio quality, perfectly adequate for educational speech content.

## Don't Hand-Roll

Problems that look simple but have existing solutions:

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| HTTP retries with backoff | Custom retry loops with time.sleep() | httpx with retry config or tenacity decorator | Exponential backoff with jitter, status code filtering, retry storms prevention, battle-tested |
| Content hashing | MD5 or custom hash | hashlib.sha256() first 16 chars | Collision-resistant, stdlib, consistent with security best practices |
| Audio file duration calculation | Parsing MP3 headers manually | mutagen library or simple formula (filesize * 8 / bitrate) | MP3 headers complex, mutagen handles all formats, formula sufficient for estimation |
| API authentication | Custom header builders | Official SDK client initialization | SDKs handle auth, token refresh, API versioning automatically |
| Path manipulation | String concatenation with os.sep | pathlib.Path | Cross-platform, readable, prevents path bugs |
| JSON serialization of Pydantic models | json.dumps(model.__dict__) | model.model_dump() + json.dumps() | Handles nested models, date serialization, enum conversion correctly |

**Key insight:** TTS APIs have complex edge cases (rate limiting, token limits, encoding issues, API version changes). Official SDKs abstract these away. Custom HTTP clients miss subtle issues that cause production failures.

## Common Pitfalls

### Pitfall 1: Rate Limiting Without Backoff

**What goes wrong:** TTS APIs return 429 (Too Many Requests) errors when generating audio for many sentences rapidly, causing generation to fail.

**Why it happens:** Both Google Cloud TTS and OpenAI TTS have rate limits (requests per minute). Naive loops hit limits immediately.

**How to avoid:** Implement exponential backoff with jitter. Retry on 429 and 5xx errors only, not 4xx client errors.

**Warning signs:** Consistent failures after first N sentences, "429 Too Many Requests" in error logs.

**Example:**
```python
# Use httpx with retry configuration
import httpx
from httpx import HTTPStatusError

transport = httpx.HTTPTransport(retries=3)
client = httpx.Client(transport=transport, timeout=30.0)

# Or use tenacity for more control
from tenacity import retry, stop_after_attempt, wait_exponential, retry_if_exception_type

@retry(
    stop=stop_after_attempt(3),
    wait=wait_exponential(multiplier=1, min=2, max=30),
    retry=retry_if_exception_type(HTTPStatusError),
)
def call_tts_api(text: str) -> bytes:
    # TTS API call here
    pass
```

### Pitfall 2: Forgetting Content Hash for Incremental Generation

**What goes wrong:** Pipeline regenerates all audio on every run, wasting API costs even when sentences haven't changed.

**Why it happens:** Manifest tracks file existence but not sentence content, so changed sentences aren't detected.

**How to avoid:** Store content_hash (SHA-256 of sentence text) in manifest entry. Compare hash before regenerating.

**Warning signs:** Audio generation takes full time on every pipeline run, API costs don't decrease with incremental updates.

### Pitfall 3: Using Stereo for Voice Audio

**What goes wrong:** Audio files are 2x larger than necessary (30KB instead of 15KB per sentence).

**Why it happens:** Default TTS settings often output stereo, but voice content is mono-equivalent (same left/right channels).

**How to avoid:** Explicitly configure mono output or convert to mono post-generation. For Google Cloud TTS, default is mono. For OpenAI, check if stereo and convert if needed.

**Warning signs:** Audio files larger than expected (~30KB vs ~15KB per sentence).

### Pitfall 4: Plain Text with Special Characters

**What goes wrong:** Numbers, URLs, abbreviations, currency symbols read incorrectly (e.g., "$1,000,000" as "dollar one comma zero zero zero...").

**Why it happens:** TTS engines use context-dependent text normalization. Some patterns are ambiguous without markup.

**How to avoid:** Start with plain text. If specific sentences sound wrong, use SSML `<say-as>` tags for problematic patterns. Don't over-engineer with SSML everywhere.

**Warning signs:** User reports of strange pronunciation for numbers, dates, abbreviations.

### Pitfall 5: Missing API Key Detection Too Late

**What goes wrong:** Pipeline starts Pass 6, loads sentences, then fails when initializing TTS client with "API key not found" error.

**Why it happens:** API key check happens during client initialization, not at pass start.

**How to avoid:** Check environment variable exists BEFORE instantiating AudioGenerator. Print warning and skip Pass 6 entirely if missing.

**Warning signs:** Pipeline crashes midway through Pass 6 instead of gracefully skipping.

**Example:**
```python
# In run_all.py
audio_api_key = os.environ.get("GOOGLE_TTS_API_KEY")
if not audio_api_key:
    print("  WARNING: GOOGLE_TTS_API_KEY not set — skipping audio generation")
else:
    generator = AudioGenerator(config, api_key=audio_api_key, ...)
```

### Pitfall 6: Hardcoded Language/Voice Parameters

**What goes wrong:** When adding new language pack (e.g., French), must modify code to change voice parameters.

**Why it happens:** Language code and voice gender hardcoded in audio_generator.py instead of reading from config.

**How to avoid:** Read language_code from config.languages.target_code, read voice gender from config.audio_generation.voice_gender (default "male").

**Warning signs:** Config has languages.target="French" but audio still generates Spanish, need code changes for new languages.

### Pitfall 7: Not Handling Long Sentences

**What goes wrong:** TTS API rejects sentences over character limit (e.g., OpenAI has 4096 character limit for input text).

**Why it happens:** Pipeline sentences occasionally exceed limits due to complex grammar explanations or long contextual examples.

**How to avoid:** Check text length before API call. If over limit, either split sentence (risky for context) or skip with error log. Most sentences in language learning content are under 200 chars, so edge case.

**Warning signs:** Random API failures with "input too long" errors for certain sentences.

```python
# Source: Audio file size calculation formula from web search
def estimate_audio_size(text: str, bitrate_kbps: int = 64) -> int:
    """
    Estimate MP3 file size in bytes.

    Args:
        text: Input text to synthesize
        bitrate_kbps: Audio bitrate (default 64kbps for voice)

    Returns:
        Estimated file size in bytes

    Formula: filesize = (bitrate * duration) / 8
    Duration estimate: ~150 words per minute for Spanish TTS
    """
    words = len(text.split())
    duration_seconds = (words / 150) * 60  # 150 words/min

    # Mono channel
    channels = 1

    # File size in bytes: (bitrate_kbps * 1000 * duration * channels) / 8
    size_bytes = (bitrate_kbps * 1000 * duration_seconds * channels) / 8

    return int(size_bytes)
```

## State of the Art

| Old Approach | Current Approach | When Changed | Impact |
|--------------|------------------|--------------|--------|
| gTTS (free Google TTS) | Google Cloud TTS official SDK | Always production standard | gTTS unofficial, no voice control, no production SLA; official SDK required for quality |
| OpenAI TTS-1 | OpenAI gpt-4o-mini-tts | Late 2025 | Newest model more reliable, better quality, same API |
| requests library | httpx library | 2023+ | httpx has better async support, built-in retry, modern API |
| Service account auth (JSON file) | API key auth | 2025+ | API keys simpler for TTS, no file management, easier CI/CD |
| SSML by default | Plain text first, SSML only when needed | Best practice 2024+ | SSML adds complexity, modern TTS handles normalization well, add only for edge cases |
| 128kbps for all audio | 64kbps for voice-only | Always best practice | Voice is not music, 64kbps is AM radio quality (perfectly clear), half the file size |

**Deprecated/outdated:**
- **pyttsx3:** Offline TTS library, robotic voice quality, not suitable for educational content
- **Festival TTS:** Legacy open-source TTS, poor quality compared to neural TTS APIs
- **Amazon Polly Standard voices:** Neural voices (introduced 2019) are now standard, Standard voices sound robotic

   - What we know: Google Cloud TTS outputs MP3, 64kbps is optimal for voice
   - What's unclear: Whether Google Cloud TTS API exposes bitrate parameter (OpenAI doesn't)
   - Recommendation: Generate sample audio, check actual bitrate with `mutagen` or `ffprobe`. If higher than 64kbps, post-process with ffmpeg to reduce size. Likely default is acceptable.

   - What we know: Image assets use `require('../../assets/images/cards/ch01_s01.webp')` pattern successfully
   - What's unclear: Whether MP3 files work identically to images in Metro bundler static requires
   - Recommendation: Assume same pattern works (high confidence based on Metro docs supporting audio assets). Validate during implementation. Fallback: use Expo FileSystem if require() issues arise.

4. **API rate limits specifics**
   - What we know: Both Google Cloud TTS and OpenAI TTS have rate limits, need exponential backoff
   - What's unclear: Exact requests-per-minute limits for each provider (varies by tier/plan)
