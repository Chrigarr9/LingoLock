# Phase 02.3 Plan 01: Audio Generation Module Summary

**One-liner:** Multi-provider TTS audio generator with Google Cloud TTS (primary) and OpenAI TTS (secondary), manifest-based incremental generation using content hashes, config-driven language selection.

## What Was Built

Created the core audio generation module for the content pipeline (Pass 6), mirroring the image generation architecture but generating MP3 audio files from sentence text using TTS APIs.

**AudioGenerator class features:**
- Multi-provider support: Google Cloud TTS and OpenAI TTS
- Manifest-based incremental generation with content hash comparison
- Automatic skip of unchanged sentences on re-runs
- Retry logic with exponential backoff (max 3 retries)
- Failed API calls produce failed manifest entries without crashing
- Config-driven provider and language selection

**Config and models:**
- AudioGenerationConfig: provider, voice_gender, speaking_rate
- AudioManifestEntry: file, status, error, content_hash
- AudioManifest: provider, language, audio dict
- Updated deck config YAML with audio_generation section

## Implementation Details

### AudioGenerator Architecture

**Pattern mirrored from ImageGenerator:**
- Constructor with config, api_key, output_base, max_retries
- `_deck_dir()` returns output path based on deck ID
- `_sentence_key()` generates consistent ch{NN}_s{NN} format
- `_content_hash()` generates 16-char SHA-256 hash for deduplication

**Provider methods:**
- `_generate_google()`: Google Cloud TTS with language-specific voice
- `_generate_openai()`: OpenAI TTS with gpt-4o-mini-tts model
- Lazy client initialization to enable mocking in tests

**Core generation flow:**
1. Load existing manifest (if present) for resumability
2. For each sentence: compute content hash
3. Skip if unchanged in manifest (same hash, file exists)
4. Generate audio via selected provider with retry logic
5. Write MP3 to audio/{key}.mp3
6. Create manifest entry (success or failed)
7. Write updated audio_manifest.json

**Error handling:**
- Retry loop catches exceptions, retries up to max_retries
- Exponential backoff: 2 * (attempt + 1) seconds
- Failed generations return failed manifest entry with error message
- No pipeline crash on individual sentence failures

## Decisions Made

**Provider selection:**
- Google Cloud TTS as primary: better Spanish voice quality, reliable
- OpenAI TTS as secondary: simple API, already integrated
- Config-driven selection via `audio_generation.provider` field

**Manifest format:**
- Same pattern as image manifest: dict keyed by ch{NN}_s{NN}
- Content hash (16-char SHA-256) for deduplication
- Status field: "success" or "failed"
- Error field for debugging failed generations

**Language handling:**
- Read from `config.languages.target_code` (not hardcoded)
- Google TTS accepts short codes like "es", auto-selects dialect
- Enables multi-language pipeline without code changes

**Audio format:**
- MP3 chosen for universal browser support
- Good compression ratio for mobile apps
- Native support in both TTS providers
