Modify `app/_layout.tsx` to add two web-specific features:

**1. Service worker registration (useEffect, web-only):**
Add a `useEffect` at the top of the `RootLayout` component that registers the service worker on web:

```typescript
import { Platform, View, useColorScheme } from 'react-native';
import { useEffect } from 'react';

// Inside RootLayout:
useEffect(() => {
  if (Platform.OS === 'web' && 'serviceWorker' in navigator) {
    navigator.serviceWorker.register('/sw.js').catch((err) => {
      console.warn('[SW] Registration failed:', err);
    });
  }
}, []);
```

- Only runs on web (Platform.OS check)
- Only runs if browser supports service workers
- `.catch()` prevents unhandled promise rejection on browsers that block SW
- Empty dependency array — runs once on mount

**2. Responsive container (web-only wrapper):**
Wrap the `<Stack>` in a centered max-width container on web. On native, render Stack directly (no wrapper overhead).

```typescript
const content = (
  <Stack>
    {/* existing Stack.Screen definitions unchanged */}
  </Stack>
);

return (
  <PaperProvider theme={theme}>
    {Platform.OS === 'web' ? (
      <View style={{ flex: 1, alignItems: 'center', backgroundColor: theme.colors.background }}>
        <View style={{ flex: 1, width: '100%', maxWidth: 480 }}>
          {content}
        </View>
      </View>
    ) : (
      content
    )}
  </PaperProvider>
);
```

Key details:
- Outer View centers the inner container horizontally with `alignItems: 'center'`
- Inner View constrains width to 480px max, takes full height with `flex: 1`
- Outer View uses `theme.colors.background` so areas outside the 480px column match the theme
- On native (iOS/Android), no wrapper is rendered — zero impact on native performance
- The 480px matches user decision for "looks like a phone app" on desktop
- All existing Stack.Screen definitions remain completely unchanged

**Import changes:**
- Add `Platform, View` to the `react-native` import
- Add `useEffect` to a React import (or from 'react')
- Keep all existing imports unchanged
