### Gap Analysis

#### build:web Script — Intentional Deviation (Warning, Not Blocker)

**Plan spec** (02.1-02): `"build:web": "npm run build:content && npx expo export --platform web"`

**Actual state**: `"build:web": "npx expo export --platform web"` (commit aa0c049 removed `build:content`)

**Reason (from commit message):** "Pipeline output directory isn't in the repo — bundle.ts is already committed so content regeneration is unnecessary in CI."

**Assessment:** This deviation is sound. The `spanish-content-pipeline/` Python pipeline is not installed in CI environments, and `src/content/bundle.ts` is committed to the repo (verified: file exists at 50,043 bytes, last committed in `bbd5ca9`). Running `build:content` on Vercel would fail because the pipeline dependencies are not available. The content bundle is static and regenerated locally when content changes.

**Impact on goal:** The build:web script produces a correct static export for Vercel deployment. The content chaining goal ("ensure fresh content in every web build") is achieved locally via manual `build:content` runs before committing `bundle.ts`. The Vercel build does not need it.

**Severity:** Warning (documented deviation with valid rationale, does not block deployment).

Phase 02.1 has achieved its core technical goal. All code artifacts are present, substantive, and correctly wired:
