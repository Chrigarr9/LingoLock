# Phase 2.1: PWA Deployment & Content Integration - Context

**Gathered:** 2026-03-03
**Status:** Ready for planning

<domain>
## Phase Boundary

Deploy LingoLock as a Progressive Web App hosted on Vercel with the existing build-time content bundle. Web users get full FSRS scheduling, streaks, and stats via localStorage. Mobile-first layout adapts for desktop via max-width container. No new features — same app, accessible via browser.

</domain>

<decisions>
## Implementation Decisions

### Hosting & Access
- Host on **Vercel** with auto-deploy on push to master (GitHub integration)
- Default Vercel URL for now (e.g., `lingolock.vercel.app`) — no custom domain yet
- Build chain: `build:content` → `expo export:web` so content bundle is always fresh from pipeline output
- Preview deploys on PRs for testing

### Web Storage
- **Full parity** with native — FSRS scheduling, streaks, stats all persist on web
- **Platform adapter pattern** — storage interface that uses MMKV on native and localStorage on web; service layer code (`storage.ts`, `statsService.ts`, `fsrs.ts`) stays the same
- Local-only storage, no cross-device sync

### PWA Features
- Service worker for offline caching (app shell + content bundle work after first load)
- Web app manifest for "Add to Home Screen" install prompt
- Manifest uses existing brand: name "LingoLock", theme color #FFA056, existing icon assets

### Responsive Layout
- Max-width container (~480px centered) on desktop — looks like a phone app, no full desktop redesign
- Mobile web works as-is since UI is already mobile-first (SafeAreaView, fixed layout)

### Claude's Discretion
- Service worker caching strategy (cache-first vs network-first)
- Exact localStorage key structure
- Web-specific polyfills or shims needed for react-native-web compatibility
- Vercel configuration details (vercel.json, build commands)

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `src/content/bundle.ts`: Auto-generated content bundle with ~500+ cloze cards — works on web as-is (pure TypeScript)
- `scripts/build-content.ts`: Content pipeline transform — can run in Vercel build step
- `src/theme/index.ts`: MD3 theme with light/dark mode + custom tokens — already has web-compatible `backdropFilter` handling
- `src/components/*`: All UI components use react-native-paper — generally web-compatible via react-native-web

### Established Patterns
- **Storage**: All persistence goes through `src/services/storage.ts` using MMKV — this is the single file that needs a platform adapter
- **Stats**: `statsService.ts` and `fsrs.ts` read/write via storage.ts — no direct MMKV calls
- **Routing**: Expo Router with file-based routes (`app/` directory) — works on web out of the box
- **Theming**: `useAppTheme()` hook + PaperProvider — web-compatible

### Integration Points
- `app.json` web config: already has `"bundler": "metro"`, `"output": "static"` — foundation for Vercel deploy
- `package.json` scripts: needs new `build:web` script chaining `build:content` + `expo export:web`
- `src/services/storage.ts`: Primary integration point for platform adapter (MMKV → localStorage)
- `react-native-mmkv`: Native module — needs conditional import or platform-specific file (`.web.ts`)

</code_context>
