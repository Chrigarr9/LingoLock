# Phase 02.1 Plan 01: Web Storage Adapter Summary

**One-liner:** localStorage-backed storage adapter mirroring MMKV API via Metro `.web.ts` platform override.

## What Was Built

Created `src/services/storage.web.ts` — a drop-in web replacement for the native MMKV storage service. Metro bundler automatically resolves `.web.ts` over `.ts` for web builds, so all existing service-layer code (statsService, fsrs, cardSelector) continues working on web without any modifications.

### Exported Functions

All 9 functions match the native `storage.ts` API exactly:

| Function | localStorage Key | Behaviour |
|---|---|---|
| `saveCardState(id, state)` | `ll.card.{id}` | JSON serialises CardState |
| `loadCardState(id)` | `ll.card.{id}` | Returns null for missing keys |
| `loadAllCardStates()` | `ll.card.*` | Iterates all keys, filters by prefix |
| `deleteCardState(id)` | `ll.card.{id}` | removeItem, no-throw on missing |
| `saveStats(stats)` | `ll.stats` | JSON serialises PersistedStats |
| `loadStats()` | `ll.stats` | Returns DEFAULT_STATS spread copy when empty |
| `saveAudioMuted(bool)` | `ll.audio_muted` | Stores 'true'/'false' string |
| `loadAudioMuted()` | `ll.audio_muted` | Returns false when not set |
| `clearAllData()` | `ll.*` | Removes all ll. prefixed keys |
