**Create `src/services/storage.ts`** — MMKV persistence layer:

```typescript
/**
 * MMKV storage service for LingoLock
 * Two separate storage instances: card FSRS states and stats/progress
 *
 * MMKV v4 uses createMMKV() factory (NOT new MMKV() constructor).
 * All reads are synchronous — no async loading delays on challenge screen start.
 */
import { createMMKV } from 'react-native-mmkv';

// Singleton storage instances
export const cardStorage = createMMKV({ id: 'lingolock.cards' });
export const statsStorage = createMMKV({ id: 'lingolock.stats' });
```

Add these functions to storage.ts:

1. **`saveCardState(cardId: string, state: CardState): void`** — Serializes CardState to JSON string and stores in cardStorage with key = cardId.

2. **`loadCardState(cardId: string): CardState | null`** — Reads from cardStorage, parses JSON. Returns null if not found. CRITICAL: Date fields (due, last_review) are stored as ISO strings — they stay as strings in CardState (the FSRS service handles Date reconstruction when needed).

3. **`loadAllCardStates(): CardState[]`** — Gets all keys from cardStorage, loads each one, returns array. Used by stats computation and card selector.

4. **`deleteCardState(cardId: string): void`** — Removes a card state entry.

5. **`saveStats(stats: PersistedStats): void`** — Serializes PersistedStats to JSON, stores in statsStorage with key 'stats'.

6. **`loadStats(): PersistedStats`** — Loads and parses stats from statsStorage. Returns default stats object if not found: `{ currentStreak: 0, lastSessionDate: null, totalCorrect: 0, totalAnswered: 0, perAppStats: {} }`.

7. **`clearAllData(): void`** — Clears both storages. For debugging/testing.

Import types from `../types/vocabulary` — use the CardState and PersistedStats interfaces. If those types don't exist yet (02-01 hasn't run), define them inline temporarily and mark with a TODO comment.

**Create `src/services/fsrs.ts`** — FSRS scheduling wrapper:

```typescript
/**
 * FSRS spaced repetition scheduler for LingoLock
 * Wraps ts-fsrs library with app-specific logic:
 * - Answer type graduation based on stability (MC2 → MC4 → Text)
 * - Card mastery definition for chapter unlock
 * - Serialization helpers for MMKV storage
 */
import { createEmptyCard, fsrs, generatorParameters, Rating, State, Card } from 'ts-fsrs';
```

FSRS configuration (from research):
```typescript
const params = generatorParameters({
  request_retention: 0.9,   // Target 90% retention
  maximum_interval: 365,     // Max 1 year between reviews
  enable_fuzz: true,         // Add randomness to intervals
  enable_short_term: false,  // No learning steps — wrong answers handled by session queue
});
const scheduler = fsrs(params);
```

Create the scheduler instance ONCE at module level (not per render). Export these functions:

1. **`createNewCardState(cardId: string): CardState`** — Creates a new FSRS empty card via `createEmptyCard()`, converts to CardState format with ISO date strings.

2. **`scheduleReview(cardState: CardState, correct: boolean): CardState`** — Reconstructs ts-fsrs Card from CardState (converting ISO strings back to Date objects), calls `scheduler.next(card, new Date(), correct ? Rating.Good : Rating.Again)`, converts result back to CardState format. Returns updated CardState ready for MMKV storage.

3. **`getAnswerType(cardState: CardState | null): 'mc2' | 'mc4' | 'text'`** — Determines answer type from FSRS stability. Null (new card) returns 'mc2'. Thresholds per user decision (Claude's discretion):
   - stability < 1.5 → 'mc2' (first 1-2 reviews, 50% baseline chance)
   - stability < 4.0 → 'mc4' (intermediate, 25% baseline chance)
   - stability >= 4.0 → 'text' (mastery, pure recall)

4. **`isCardMastered(cardState: CardState): boolean`** — Returns true if card state equals `State.Review` (card has survived at least one review cycle). Used for chapter mastery percentage.

5. **`isDue(cardState: CardState): boolean`** — Returns true if `new Date(cardState.due) <= new Date()`. Card is due for review.

6. **Helper: `toFSRSCard(cardState: CardState): Card`** — Internal helper converting CardState (with ISO strings) to ts-fsrs Card (with Date objects). CRITICAL to avoid the Date serialization pitfall.

7. **Helper: `fromFSRSCard(cardId: string, card: Card): CardState`** — Internal helper converting ts-fsrs Card back to CardState with ISO strings.

**Anti-patterns to avoid:**
- Do NOT create a new `fsrs()` instance per function call — use the module-level singleton.
- Do NOT store Date objects in MMKV — always serialize to ISO strings.
- Do NOT use `Rating.Easy` — only `Rating.Good` (correct) and `Rating.Again` (incorrect) per CONTEXT decisions.
