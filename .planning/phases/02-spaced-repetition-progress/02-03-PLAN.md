  <name>Task 1: Card Selector — Session Building & Wrong-Answer Reinsertion</name>
  <files>src/services/cardSelector.ts, src/services/cardSelector.test.ts</files>
  <action>
**buildSession(cardCount: number, sourceApp?: string): SessionCard[]**
- Returns exactly `cardCount` session cards
- Priority order: due review cards first, fill remaining with new words from current chapter
- ALWAYS includes at least 1 new word (even if enough due reviews exist, replace last due with a new word)
- New word = card with no CardState in MMKV (never seen before)
- Due review = card with CardState where isDue() returns true
- Each SessionCard has answerType determined by getAnswerType(cardState) and choices generated for MC modes
- MC choices: shuffle correct word + N distractors from card.distractors array (1 for MC2, 3 for MC4)
- If not enough due reviews AND not enough new words in current chapter, use new words from next unlocked chapter
- If total available cards < cardCount, return as many as available

**handleWrongAnswer(queue: SessionCard[], currentIndex: number, wrongCard: SessionCard): SessionCard[]**
- Re-inserts wrongCard into queue ~3-5 positions after currentIndex
- Exact position: min(currentIndex + 4, queue.length) — inserts AT that index
- Returns new queue array (immutable)
- Does NOT increase the total card count for progress dots (the re-inserted card is a bonus)

**getCurrentChapter(): number**
- Returns the chapter number the user is currently working on
- Scans CHAPTERS in order: first chapter where mastery < 80% is the current chapter
- If all chapters >= 80%, returns the last chapter (review mode)
- Mastery = percentage of chapter cards where isCardMastered(loadCardState(card.id)) is true

Test cases:
- buildSession(5) with 0 due reviews returns 5 new words from chapter 1
- buildSession(5) with 3 due reviews returns 3 due + 2 new (including at least 1 new)
- buildSession(5) with 10 due reviews returns 4 due + 1 new (always 1 new guaranteed)
- buildSession(5) with 0 due and 3 new in ch1 (ch2 unlocked) returns 3 from ch1 + 2 from ch2
- handleWrongAnswer inserts card at position currentIndex + 4
- handleWrongAnswer at end of queue appends card at queue.length
- MC2 card has 2 choices (correct + 1 distractor), shuffled
- MC4 card has 4 choices (correct + 3 distractors), shuffled
- Text card has no choices
- getCurrentChapter returns 1 when no cards reviewed
- getCurrentChapter returns 2 when chapter 1 >= 80% mastered

Create `src/services/cardSelector.ts` implementing the three functions above. For tests, mock the storage and FSRS service imports. Use Jest (or whatever test runner is configured — check package.json; if none, set up Jest with ts-jest).

For MC choice generation: shuffle the choices array using Fisher-Yates shuffle so the correct answer isn't always in the same position.

For the "at least 1 new word" guarantee: after building the initial queue from due reviews, if all slots are taken by due reviews, replace the LAST due review with a new word.

  <files>src/services/statsService.ts, src/services/statsService.test.ts</files>
  <action>
**updateStatsAfterSession(correctCount: number, totalCount: number, sourceApp: string): void**
- Loads current stats from MMKV
- Updates totalCorrect += correctCount, totalAnswered += totalCount
- Updates streak: if lastSessionDate is today, no change. If lastSessionDate is yesterday, increment streak. If lastSessionDate is older (or null), reset streak to 1.
- Sets lastSessionDate to today (ISO date string, date portion only: YYYY-MM-DD)
- Updates perAppStats[sourceApp]: increments sessions by 1, cards by totalCount
- Saves updated stats to MMKV

**getStreak(): number**
- Loads stats, returns currentStreak
- If lastSessionDate is today or yesterday, streak is valid
- If lastSessionDate is older than yesterday, streak is 0 (stale — broken but not yet updated)

**getSuccessRate(): number**
- Returns Math.round((totalCorrect / totalAnswered) * 100) or 0 if totalAnswered is 0

**getChapterMastery(chapterNumber: number): number**
- Gets cards for the chapter from content bundle
- For each card, loads CardState and checks isCardMastered
- Returns Math.round((mastered / total) * 100)

**getCardsDueCount(): number**
- Loads all card states, filters by isDue(), returns count

**getCurrentChapterNumber(): number**
- Delegates to cardSelector.getCurrentChapter() — re-exported for convenience

Test cases:
- updateStatsAfterSession: first session sets streak to 1 and lastSessionDate to today
- updateStatsAfterSession: second session same day keeps streak at 1
- updateStatsAfterSession: session next day increments streak to 2
- updateStatsAfterSession: session after 2-day gap resets streak to 1
- getSuccessRate: 3 correct out of 5 returns 60
- getSuccessRate: 0 answered returns 0
- getChapterMastery: 4 of 5 cards mastered returns 80
- getChapterMastery: 0 cards mastered returns 0
- getCardsDueCount: returns count of cards where isDue is true
- perAppStats: tracks sessions and cards per source app

Create `src/services/statsService.ts` implementing the functions above. For date comparison in streak logic, compare ISO date strings (YYYY-MM-DD format) — no need for a date library, just compare string values after extracting the date portion.

For tests, mock storage.ts (loadStats, saveStats) and fsrs.ts (isCardMastered, isDue) to isolate pure logic.

**Important:** The "stale streak" case (lastSessionDate older than yesterday) should return the stored streak value from getStreak() — let updateStatsAfterSession handle the reset. This avoids the UI showing 0 mid-session before the update runs.

Actually, reconsider: getStreak() should return the ACTUAL current streak. If lastSessionDate is 3 days ago, the streak IS 0. The UI should reflect reality. updateStatsAfterSession resets and increments to 1 when a new session happens.
