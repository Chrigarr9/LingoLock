**Key changes from Phase 1:**

1. **Session initialization:** Replace `PLACEHOLDER_CARDS.slice(0, cardCount)` with `buildSession(cardCount, params.source)`. This returns `SessionCard[]` with FSRS-scheduled due reviews + new words.

2. **Card display:** Replace `<VocabularyCard>` with `<ClozeCardDisplay>`. Replace the existing MC question text with the cloze sentence from the card.

3. **Answer handling — all three modes in one flow:**

   For TEXT mode (answerType === 'text'):
   - Show `<AnswerInput>` below the cloze card
   - On submit: validate answer against `sessionCard.card.wordInContext` using existing `validateAnswer` from answerValidation.ts
   - If correct: green feedback, schedule FSRS review (correct), save card state, show reveal, auto-advance after 1.5s
   - If incorrect: red feedback, schedule FSRS review (incorrect), save card state, show reveal, show "Next" button AND re-insert card into queue via handleWrongAnswer

   For MC2/MC4 mode (answerType === 'mc2' or 'mc4'):
   - Show `<MultipleChoiceGrid>` with sessionCard.choices
   - correctAnswer = sessionCard.card.wordInContext (the Spanish word)
   - On select: same correct/incorrect flow as text mode

4. **Wrong-answer reinsertion:**
   When answer is incorrect:
   - Call `handleWrongAnswer(queue, currentIndex, wrongSessionCard)` to get new queue
   - Update the session queue state
   - The re-inserted card does NOT increase the total for ProgressDots — track the "original count" separately from "queue length"
   - The total shown in progress is the initial cardCount from URL params

5. **Session completion:**
   - When all original cards answered correctly (currentIndex reaches original cardCount), show completion screen
   - Call `updateStatsAfterSession(correctCount, cardCount, params.source)` to update streak/stats
   - Show score and ContinueButton (reuse existing component)

6. **Post-answer reveal:**
   - After answering (correct or incorrect), show `<AnswerReveal>` below the card
   - For correct answers: auto-advance after 1.5 seconds (show reveal during this time)
   - For incorrect answers: reveal stays visible until user taps "Next"

7. **Progress tracking:**
   - `ProgressDots` total = original cardCount (not queue length which may grow from wrong answers)
   - Progress label: "CHAPTER {getCurrentChapter()} · CARD {currentIndex + 1} OF {cardCount}"
