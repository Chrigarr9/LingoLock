    1. **WidgetCardData** interface:
       ```typescript
       export interface WidgetCardData {
         cardId: string;
         sentence: string;       // Cloze sentence with _____ gap
         germanHint: string;     // German hint word
         correctAnswer: string;  // wordInContext
         imageUri?: string;      // Optional card image
         cardsLeft: number;      // Total cards remaining in session
         streakCount: number;    // Current streak
       }
       ```

    2. **getWidgetCardData()** function:
       - Import getDueCards logic from cardSelector (reuse the due-card scanning logic)
       - Scan all CHAPTERS for cards with stored state where isDue() is true (same as buildSession's due card logic)
       - Per user decision on widget content filtering:
         - If images supported: show both new + repetition cards (full FSRS queue)
         - If images NOT supported: only repetition cards (seen at least once = has CardState)
         - Start with repetition-only for Lock Screen (conservative approach)
       - Get streak from statsService.getStreak()
       - Return WidgetCardData for the first due card, or null if no cards due
       - Also return cardsLeft count

    3. **updateWidgetData()** function:
       - Call getWidgetCardData() to get current card
       - If expo-widgets provides an updateTimeline/updateSnapshot API, call it with the card data
       - This function should be called after every answer (in-app, notification, or widget) to keep widget fresh
       - If expo-widgets API not available at runtime (web, or pre-SDK 55), silently no-op

    4. **clearWidgetData()** function:
       - Clear widget state when user completes all cards
       - Widget should show empty state

    - `getWidgetCardData()` returns null
    - `updateWidgetData()` returns void
    - `clearWidgetData()` returns void
